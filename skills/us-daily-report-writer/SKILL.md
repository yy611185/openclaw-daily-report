## 前置依赖
必须确保以下技能已执行完毕并有输出：
1. ✅ `us-daily-market-data` → 市场数据
2. ✅ `us-daily-news-brief` → 新闻与情绪数据
3. ✅ `us-daily-equity-research` → 因子分析与选股结果

如果任一前置数据缺失，必须先调用对应技能获取。

## 🔁 自检循环（SELF-REFLECTION LOOP）

报告生成完毕后，**必须**执行自检循环，最多执行2次。

### 循环 Step 1：自我审查（Critic Mode）

逐项检查以下问题：

| 检查项 | 具体要求 |
|--------|---------|
| 数据真实性 | 所有数据是否来自 yfinance？是否存在编造数据？ |
| 数据使用 | 是否引用了未在前置步骤中获取的数据？ |
| 逻辑一致性 | 市场总览的判断是否与板块分析、个股分析一致？ |
| 完整性 | 7个章节是否完整？Top 5 是否每只都有三因子分析？ |
| 模糊用词 | 是否存在"可能"、"大概"等禁止词汇？ |
| 风险覆盖 | 是否遗漏了重要的板块或系统性风险？ |
| 结论支撑 | 每个结论是否有明确的数据或新闻支撑？ |

输出自检结果：
```json
{
  "check_round": 1,
  "errors": [
    {
      "section": "4. 重点个股",
      "issue": "NVDA 的 PE 数据未从 yfinance 获取",
      "severity": "high"
    }
  ],
  "total_errors": 1,
  "max_severity": "high"
}
```

### 循环 Step 2：纠错（Fix Mode）

如果存在 errors：
- **定位**：找到报告中的问题段落
- **修复**：重写该段落，确保数据来源正确、逻辑自洽
- **禁止**：仅修改表面文字而不修复根本问题

### 循环 Step 3：再验证（Verify Mode）

修复后重新执行 Step 1 的所有检查项：
- 确认所有问题已解决
- 确认未引入新问题
- 确认报告结构完整

### 终止条件

```
IF errors == [] AND total_errors == 0:
    → 输出 "✅ FINAL APPROVED"
    → 将自检结果附加到报告附录
ELSE IF check_round < 2:
    → 返回 Step 1 再循环一次
ELSE:
    → 输出 "⚠️ CONDITIONAL APPROVED"
    → 标注未解决问题
    → 将自检结果附加到报告附录
```

## 输出要求
- 报告使用 Markdown 格式
- 文件命名：`report_YYYY-MM-DD.md`
- 所有数据保留2位小数
- ticker 使用大写英文
- 报告正文使用中文
